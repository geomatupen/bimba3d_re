Chapter 1 - Scope Index

The UI exposes four tuning scopes.

Scope What It Touches Risk Level Best Use
core_only Learning rates plus only grow_grad2d Low Closest to baseline behavior with light adaptation
core_individual Learning rates only (per group) Medium LR shaping when densification policy should remain stable
core_ai_optimization Learning rates plus only grow_grad2d, with adaptive improvement gate Medium Core-only behavior with smarter update timing based on recent improvement
core_individual_plus_strategy Learning rates plus full strategy set High Advanced tuning once you already match baseline early signals

Internal note: with_strategy is a legacy backend alias that is automatically normalized to core_individual_plus_strategy. It is not a separate user-facing scope.

Chapter 2 - Tuned Parameters

2.1 Learning-rate groups that are tuned

Modified mode multiplies optimizer learning rates for these groups: means, opacities, scales, quats, sh0, shN.

2.1b LR names you will see (quick mapping)

Common Name Internal Group(s) What It Controls
position_lr means 3D point/center movement in space.
opacity_lr opacities Transparency and visibility evolution.
scaling_lr scales Gaussian size changes.
rotation_lr quats Gaussian orientation updates.
sh_lr sh0 and shN Appearance/color fitting (base and high-frequency SH terms).

In logs and summaries, sh_lr is often represented by separate sh0 and shN channels, while position_lr maps to means.

2.1a Effect of changing each LR group

This section explains the training effect when these learning rates are increased or decreased. It is different from Section 2.1b, which only maps names to internal groups.

Group What It Represents If LR Is Higher If LR Is Lower
means 3D Gaussian center positions in space. Faster geometry movement and scene re-alignment. More stable geometry, slower correction of misplacement.
opacities Visibility/transparency strength of each Gaussian. Quicker visibility changes; can prune/keep differently. Smoother opacity evolution; less abrupt appearance changes.
scales Gaussian size (spatial spread/footprint). Faster resizing, can cover errors quickly but may blur. More conservative size updates and steadier detail structure.
quats Gaussian orientation (rotation via quaternions). Faster orientation correction for anisotropic splats. More stable orientation, slower directional adjustment.
sh0 Low-order spherical harmonics (base color term, diffuse body). Faster broad color/brightness adaptation. More stable base tone, slower global color correction.
shN Higher-order spherical harmonics (view-dependent/high-frequency appearance). Faster specular/detail appearance fitting; can become noisy if too high. Cleaner but slower high-frequency appearance convergence.

Practical shorthand: sh0 controls coarse color body, and shN controls finer directional appearance.

2.2 Strategy parameters that may be tuned

  • grow_grad2d: growth threshold behavior for densification sensitivity.
  • prune_opa: pruning opacity threshold.
  • refine_every: densification/refinement cadence.
  • reset_every: reset cadence tied to refinement behavior.

Scope decides which of these are allowed to move.

2.3 Timing parameters that govern adaptation

  • tune_start_step: first step where LR-side tuning is allowed.
  • tune_end_step: last step where LR-side tuning is allowed.
  • tune_interval: how often checks happen (every N steps).
  • tune_min_improvement: required relative improvement gate.
  • densify_from_iter: strategy tuning start (same as strategy window start).
  • densify_until_iter: strategy tuning end (same as strategy window end).
Chapter 3 - What Happens At The Beginning

3.1 Before tune_start_step

No modified LR adaptation is applied. Training follows base optimizer schedules. This is where initial structure and early densification patterns are formed.

3.2 Between tune_start_step and tune_end_step

LR adaptation can apply every tune_interval steps. In core_individual and core_ai_optimization, the tune_min_improvement gate is used so updates are skipped unless improvement is below the threshold.

3.3 Strategy window starts at densify_from_iter

Strategy-side updates only run in the densification window [densify_from_iter, densify_until_iter]. This means strategy can be inactive early even while LR tuning is active, depending on your start steps.

Early-signal rule: if you are not matching baseline by 3k to 5k, start with core_only and delay or narrow strategy influence.
Chapter 4 - Scope Details

4.1 core_only

  • Applies LR multipliers by profile (high_loss, mid_loss, low_loss).
  • Applies strategy only to grow_grad2d.
  • Does not change prune_opa, refine_every, reset_every.

4.2 core_individual

  • Applies LR multipliers only.
  • No strategy parameter changes.
  • Uses the improvement gate logic in the modified control path.

4.3 core_ai_optimization

  • Applies the same LR multipliers as other scopes.
  • Applies strategy only to grow_grad2d (same strategy coverage as core_only).
  • Uses the improvement gate logic (same gating family as core_individual).

4.4 core_individual_plus_strategy

  • Applies LR multipliers.
  • Applies full strategy updates: grow_grad2d, prune_opa, refine_every, reset_every.
  • Most powerful scope, also easiest to destabilize early growth if timing is aggressive.

4.5 Rule profiles used by all scopes

Profile is selected from loss level:

  • high_loss when loss >= 0.20
  • mid_loss when loss >= 0.08
  • low_loss otherwise

4.6 How values are updated

At an eligible tuning step, each tuned value is multiplied by the profile multiplier.

What clamp means: clamp(x, min, max) keeps a value inside bounds. If x is below min, result is min. If x is above max, result is max. Otherwise, result is x.

  • Example: clamp(1.2, 0.0, 1.0) = 1.0
  • Example: clamp(-0.1, 0.0, 1.0) = 0.0
  • Example: clamp(0.6, 0.0, 1.0) = 0.6
  • new_lr[group] = clamp(old_lr[group] * lr_mult[group], 1e-7, 1.0)
  • new_grow_grad2d = clamp(old_grow_grad2d * mult, 5e-5, 5e-3)
  • new_prune_opa = clamp(old_prune_opa * mult, 1e-4, 5e-2)
  • new_refine_every = clamp(int(old_refine_every * mult), 25, 300)
  • new_reset_every = clamp(int(old_reset_every * mult), max(refine_every, 1000), 6000)

4.6a Exactly what gets updated in each scope

Scope LR Parameters Updated Strategy Parameters Updated Not Updated
core_individual means, opacities, scales, quats, sh0, shN None grow_grad2d, prune_opa, refine_every, reset_every
core_only means, opacities, scales, quats, sh0, shN grow_grad2d only prune_opa, refine_every, reset_every
core_ai_optimization means, opacities, scales, quats, sh0, shN grow_grad2d only (adaptive-gated by tune_min_improvement) prune_opa, refine_every, reset_every
core_individual_plus_strategy means, opacities, scales, quats, sh0, shN grow_grad2d, prune_opa, refine_every, reset_every None (within window and apply flags)

In short: all four scopes update the same six LR groups; the difference is strategy coverage and whether tune_min_improvement gating is required.

4.8 Example update at one step

If profile is mid_loss, and current values are means_lr=1.0e-4, opacities_lr=0.05, refine_every=250, then after one update:

  • means_lr -> 9.6e-5 (multiplied by 0.96)
  • opacities_lr -> 0.046 (multiplied by 0.92)
  • refine_every -> 255 (multiplied by 1.02 and rounded to int)
Chapter 5 - Suggested Presets

5.1 Safe start (recommended first test)

  • tune_scope=core_only
  • tune_start_step=3500
  • tune_end_step=10000
  • tune_interval=200
  • tune_min_improvement=0.02
  • densify_from_iter=500
  • densify_until_iter=10000

5.2 Next test after early parity is achieved

  • Switch to core_individual
  • Move tune_start_step later (for example 5000)
  • Keep strategy stable until confidence is built

5.3 Adaptive core test

  • Use core_ai_optimization
  • Keep tune_min_improvement meaningful (for example 0.01 to 0.03)
  • Good when you want core-only strategy coverage with adaptive timing

5.4 Advanced test

  • Use core_individual_plus_strategy
  • Narrow strategy window to avoid late oscillation
  • Only after baseline-like growth is confirmed
Chapter 6 - Reading Your Logs Quickly

During debugging, inspect these fields first:

  • apply_lr and apply_strategy: confirms which side was active.
  • strategy_window and LR window values: confirms timing setup.
  • num_gaussians trend by 3k, 5k, 10k: fastest health signal.
  • lpips and sharpness at each eval checkpoint.
If gaussians plateau far below baseline before 10k, treat it as a capacity issue first, then tune scope/timing. Do not tune only final-stage LR.
