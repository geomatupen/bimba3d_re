EXIF-Based AI Optimization Modes

Three-Layer Framework for Intelligent Parameter Selection in 3D Reconstruction

1. Overview & Architecture

The EXIF-based AI optimization system uses reward-optimized bandit learning to select Gaussian Splatting (GSPLAT) training parameters. The system operates in three progressive feature-extraction layers, each adding complexity and richness to the feature space. However, features are mostly ignored during selection—parameter choices are driven by global learned preferences (bias/means) updated via reward feedback.

Architectural Clarification: This is NOT a traditional supervised regression. The system does not learn weights that map features → parameters (i.e., no y = f(x; W) function). Instead, it learns global parameter preferences (bias or means) via trial-and-error reward optimization. Features x are extracted for context, but parameter selection is mostly context-agnostic (except small heuristic bonuses in discrete mode).
Core Philosophy: Extract intrinsic properties from source imagery → Apply global learned parameter preferences (via bandit learning) → Continuously refine preferences through observed reward signals (final quality × speed trade-off).

Learning Paradigm: Discrete vs. Continuous

The system supports two fundamentally different learning approaches for parameter selection. Both leverage the same EXIF feature layers, but differ in how they model and optimize parameters.

DISCRETE

Preset-Based Selection

Approach: Select one of 4 fixed presets (conservative, balanced, geometry_fast, appearance_fast)

Model: Bias accumulator per preset + competitive credit assignment

Best for: Categorical decision making; interpretable presets; stable learning

CONTINUOUS

Continuous Bandit (Linear)

Approach: Sample continuous multiplier values for 8 parameters independently

Model: Gaussian sampling from learned means + direct mean gradient updates

Best for: Fine-grained parameter tuning; exploring continuous space; high dimensionality

3. Model Structure: Inputs, Outputs, and Predictions

A. Input Space (x)

x ∈ ℝ^D where D ∈ {25, 37, 47} depending on selected EXIF mode.

Feature Vector x:
x = [x₁, x₂, ..., x_D]ᵀ

where each x_i represents:
— Equipment/capture metadata (EXIF Mode 1)
— + Flight trajectory properties (EXIF Mode 2)
— + Scene content characteristics (EXIF Mode 3)

B. Output Space & Prediction Mechanisms

The output differs fundamentally based on learning paradigm.

Discrete Preset-Based Mode

Target y (discrete):
y ∈ {conservative, balanced, geometry_fast, appearance_fast}

Selected preset applied to base parameters:
y_applied = base_params × preset_multipliers[y]
Predictions ŷ (yhat_scores):
ŷ = [score_conservative, score_balanced, score_geometry_fast, score_appearance_fast]ᵀ

ŷ_i = bias[i] + (heuristic_bonus if i == heuristic_preset else 0)

k* = argmax_i (ŷ_i) [selected preset = highest-scoring preset]

Continuous Bandit Mode

Target y (continuous):
y = [m₁, m₂, ..., m₈] where m_j ∈ ℝ (continuous multipliers)

y ∈ {feature_lr_mult, position_lr_init_mult, scaling_lr_mult, opacity_lr_mult,
rotation_lr_mult, densify_grad_threshold_mult, opacity_threshold_mult, lambda_dssim_mult}
Predictions ŷ (yhat_scores via Gaussian sampling):
ŷ_j ~ Gaussian(μ_j, σ²_j) for each multiplier j

μ_j = learned mean for multiplier j
σ²_j = exploration_std = max(0.03, min(0.18, 0.18 / √(runs + 1)))

k* = "continuous_bandit_linear" [fixed method name; specific values sampled]

4. Training & Learning Framework

Common Foundation (Both Modes)

Regardless of learning paradigm, both modes use the same score computation pipeline based on visual metrics and runtime.

A. Score Computation (Quality, Time, Loss)

At each evaluation step, a composite score s(step) is computed from visual metrics and runtime.

Metric Normalization

Joint Normalization (shared scale with baseline):
For metric M ∈ {PSNR, SSIM, LPIPS, Loss}:

all_vals = [run_values] ⊕ [baseline_values]
normalized = (M - min(all_vals)) / (max(all_vals) - min(all_vals))

Direction:
— PSNR, SSIM: higher is better (direct normalization)
— LPIPS, Loss: lower is better (inverted: 1 - normalized)

Quality Score Composition

Quality Score q(step):
q = 0.40 × PSNR_norm + 0.30 × SSIM_norm + 0.30 × LPIPS_norm_inverted

Weights: Loss=40%, Appearance(PSNR+SSIM)=30% each, Geometry(LPIPS)=30%

Time Score

Time Score t_score(step):
t_ratio = elapsed_seconds(step) / time_reference
t_score = 1.0 - clamp(t_ratio, 0, 1)

time_reference = max(baseline_elapsed) if baseline exists, else max(run_elapsed)

Composite Score

Composite Score s(step):
s = 0.50 × loss_norm + 0.25 × q + 0.25 × t_score

Weights: Loss convergence=50%, Quality=25%, Speed=25%

B. Run-Level Metrics (s_best, s_end, s_run)

Anchor Steps:
t_best = step with minimum loss (best optimization point)
t_eval_best = first eval step ≥ t_best (closest eval after best loss)
t_end = final evaluation step (convergence point)
Run Scores:
s_best = s(t_eval_best) [quality at best loss point]
s_end = s(t_end) [quality at convergence]
s_run = max(s_best, s_end) [best achieved score]

C. Baseline Comparison & Reward Signal (Shared)

Baseline Scores (when baseline run available):
s_base_best = s_baseline(t_eval_best_aligned)
s_base_end = s_baseline(t_end_aligned)
s_base = max(s_base_best, s_base_end)
Reward Signal r:
IF baseline_comparison exists:
r = s_run - s_base [relative improvement]
ELSE:
r = s_run [absolute quality; baseline missing]

Sign Convention:
r > 0 → Run outperformed baseline (positive learning)
r ≤ 0 → Run matched or underperformed baseline

5. Learning Updates: Discrete vs. Continuous Bandits

Key Insight: Both modes are reward-optimized bandits, not regression models. The system learns which actions performed well historically, not how features predict optimal actions.

  • What IS learned: Bias/means (global parameter preferences); reward signal (empirical quality metric)
  • What is NOT learned: Feature-to-parameter mapping; weights that depend on image content
  • Selection mechanism: Choose actions based on learned preferences, with minimal feature dependence (except discrete heuristic bonus)

Mode 1: Discrete Preset-Based Bandit

A. Reward Mean Update

Exponential Moving Average:
runs = number of completed training runs
α = 1.0 / (runs + 1) [decreasing step size with experience]
reward_mean ← reward_mean + α × (r - reward_mean)

B. Bias Update Rule (Competitive Credit Assignment)

1
Compute running reward mean (exponential average)
2
Calculate delta: δ = r - reward_mean
3
Update selected preset: bias[k*] += learning_rate × δ
4
Update non-selected presets: bias[others] -= learning_rate × decay × δ
5
Clamp bias values to [-3.0, 3.0] to prevent divergence
Bias Update (gradient ascent on selected preset):
learning_rate = 0.10
decay_factor = 0.15

FOR each preset ∈ {conservative, balanced, geometry_fast, appearance_fast}:
IF preset == selected_preset k*:
bias[preset] ← clamp(bias[preset] + lr × δ, -3.0, 3.0)
ELSE:
bias[preset] ← clamp(bias[preset] - lr × decay × δ, -3.0, 3.0)
Interpretation: If reward is positive (δ > 0), the selected preset's bias increases, making it more likely to be chosen in future runs with similar imagery. Non-selected presets have their bias reduced slightly (penalty is scaled by decay factor 0.15), implementing competitive credit assignment.

Mode 2: Continuous Parameter Bandit

A. Exploration & Sampling

Exploration Standard Deviation (decays with experience):
σ = max(0.03, min(0.18, 0.18 / √(runs + 1)))

Interpretation:
— Starts at 0.18 (high exploration early)
— Floor at 0.03 (maintain minimum exploration)
— Decays as √(runs) to allow fine-tuning later
Per-Parameter Sampling:
FOR each multiplier j ∈ {1..8}:
μ_j = means[j] [learned center]
sampled_j ~ Gaussian(μ_j, σ²)
sampled_j ← clamp(sampled_j, lo_j, hi_j) [safe bounds]

B. Mean Update Rule (Bandit Policy Gradient — NOT Feature-Conditional)

1
Compute running reward mean (exponential average)
2
Calculate delta: δ = r - reward_mean
3
Compute learning rate: η = max(0.03, min(0.20, 0.20 / √(runs + 1)))
4
FOR each parameter: μ_j ← μ_j + η × δ × (action_j - μ_j) [global update, independent of features]
5
Clamp updated means to safe bounds [lo_j, hi_j]
Mean Update (Bandit algorithm, NOT regression):
η = max(0.03, min(0.20, 0.20 / √(runs + 1)))

FOR each multiplier j:
μ_j ← clamp(μ_j + η × δ × (a_j - μ_j), lo_j, hi_j)

where a_j = action sampled in this run (actual multiplier used)

Critical Insight: μ_j is GLOBAL, independent of input features x. All images (regardless of EXIF/flight/content) sample from the same Gaussian(μ_j, σ²). The system learns "what multiplier values worked well in the past" but NOT "what multiplier values work best for THIS image."
Safe Bounds per Multiplier:
feature_lr_mult: [0.5, 1.5]
position_lr_init_mult: [0.5, 1.5]
scaling_lr_mult: [0.5, 1.5]
opacity_lr_mult: [0.5, 1.5]
rotation_lr_mult: [0.5, 1.5]
densify_grad_threshold_mult: [0.7, 1.3]
opacity_threshold_mult: [0.7, 1.3]
lambda_dssim_mult: [0.7, 1.3]
Key Insight (Continuous Mode): The update rule μ += η × δ × (a - μ) is a bandit policy gradient step. If the sampled action a_j was above the mean (a_j > μ_j) and reward is positive (δ > 0), the mean shifts toward a_j.

Critical Limitation: The means μ are GLOBAL and do NOT condition on features x. All images (regardless of EXIF/flight/content) sample from the same distribution. This is fundamentally a trial-and-error bandit, not feature-conditioned learning.

7. Learning Behavior & Convergence

Test Runs (Both Modes)

When apply_update = False (test/compare-only mode), the system:

  • ✓ Extracts features x from test imagery (for context/logging only)
  • ✓ Selects/samples parameters from global learned preferences (bias or means), mostly ignoring x
  • ✓ Computes reward signal for comparison
  • ✗ Does NOT update learned preferences (bias/means)
  • ✗ Does NOT persist model changes
Use: Validation of bandit choices; testing new imagery; A/B comparison with baseline.

Note: Features x are computed but largely ignored during selection. Parameter choices driven by global learned preferences, not feature-to-parameter regression.

Training Runs (Both Modes)

When apply_update = True (training mode), the system additionally:

  • ✓ Updates global learned preferences (bias or means) based on reward signal
  • ✓ Persists updated model to disk
  • ✓ Tracks cumulative runs and reward mean
  • ✓ Applies exploration strategy (heuristic bonus in discrete; Gaussian variance decay in continuous)
  • ✗ Does NOT learn feature-conditioned parameter mappings (no y = f(x; W))
This is a key architectural limitation: The system is a context-free bandit, not a context-aware policy learner. Features improve interpretability and heuristic guidance, but do not drive parameter selection through learned weights.

Discrete Mode Convergence

Effective Learning Rate (decreases with experience):
α_reward = 1.0 / (runs + 1)

After 10 runs: α ≈ 0.091 (9% step size)
After 100 runs: α ≈ 0.010 (1% step size)
After 1000 runs: α ≈ 0.001 (0.1% step size)

Continuous Mode Convergence

Effective Learning Rate (decreases with experience):
η = max(0.03, min(0.20, 0.20 / √(runs + 1)))
σ = max(0.03, min(0.18, 0.18 / √(runs + 1)))

After 10 runs: η ≈ 0.063, σ ≈ 0.057
After 100 runs: η ≈ 0.020, σ ≈ 0.018
After 1000 runs: η ≈ 0.006, σ ≈ 0.006
Important: Neither system is a classical regression model y = f(x).

Discrete: Maintains preset-specific bias accumulators reflecting which presets historically performed well.

Continuous: Maintains per-parameter Gaussian means reflecting which parameter ranges yielded high rewards.

9. Key Design Principles (Both Modes)

1. Context-Free Bandit (NOT Regression)

The system learns which actions were historically good via reward feedback (bandit learning), NOT how features predict optimal actions (supervised regression). Model state (bias/means) is global and does NOT condition on input features x.

2. Decaying Learning Rates

Both modes use schedules that decay as 1/√(runs). Early exploration when data is sparse; fine-tuning when patterns are clear. Prevents divergence in later stages.

3. Joint Metric Normalization

When baseline exists, metrics normalized on shared (run ∪ baseline) scale to prevent misleading reward signals from independent scaling. If baseline missing, reward = s_run (absolute).

4. Minimal Feature Utilization

Features x extracted for logging/interpretability but mostly ignored in selection. Discrete: +0.002 heuristic bonus only.
Continuous: Features ignored entirely.

Result: Two lightweight, highly interpretable bandit systems that refine parameter selection based on empirical rewards. Trade-off: Simplicity/interpretability vs. feature-conditioned optimization (no learned x→y mapping).

11. Future Extensions (Path to Feature-Conditioned Learning)

  • Contextual Bandits (High Priority): Extend to μ(x) and bias(x) — learn feature-conditioned means/biases. Enable parameters to adapt to specific image characteristics. Requires more training data and feature-specific reward tracking.
  • Neural Policy Network: Replace context-free bandit with learned policy π(y | x; θ). End-to-end supervised learning from (image, parameter, reward) triplets. Requires substantial labeled dataset.
  • Hybrid Approach: Combine discrete preset selection with continuous fine-tuning (preset + perturbations). Coarse preset choice (4 options) + fine parameter adjustment (8 multipliers).
  • Multi-task Learning: Jointly predict multiple parameter vectors and preset probabilities from features. Better utilization of feature information.
  • Transfer Learning: Pre-train on large external photogrammetry dataset; fine-tune on project-specific data. Reduce data requirements for feature-conditioned learning.
  • Uncertainty Quantification: Predict confidence intervals on parameter selections (Bayesian approaches). Flag ambiguous scenarios; escalate to human review when confidence low.
